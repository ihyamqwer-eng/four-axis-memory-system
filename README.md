# Four-Axis Memory System

一个将「记忆」拆解为**时间、状态、项目、对话**四个维度的个人 AI 知识管理系统。

不是向量数据库的替代品——而是它的上层组织框架。

## 核心思想

现有记忆系统的问题：纯向量检索丢失时间顺序，MemGPT 上下文有限，图数据库对个人场景太重，单层日志检索效率低。

四轴按**用途**分离记忆：

| 轴 | 用途 | 类比人脑 |
|---|---|---|
| **总轴** (Timeline) | 时间索引，记录"发生了什么" | 回忆"那天发生了什么事" |
| **上帝轴** (God) | 实时状态，记录"我是谁" | "我现在穿着什么，什么心情" |
| **对象轴** (Object) | 任务/项目跟踪 | "那个事进展到哪了" |
| **会话轴** (Session) | 对话原文存档 | "他当时原话怎么说的" |

## 三层架构

```
第一层：四轴（原始数据层）
  总轴 · 上帝轴 · 对象轴 · 会话轴
         ↓
第二层：语义检索（LanceDB + Embedding）
  extract-fact → 向量化 → 语义搜索
         ↓
第三层：知识库（Wiki Vault）
  concepts · syntheses · entities · sources
```

## 设计原则

1. **只追加，不修改** — 历史可追溯
2. **按职责分离，按需检索** — 不把问题压在一个维度
3. **本地优先，API 增强** — 文件系统即可运行
4. **自动沉淀，由点及面** — 知识从对话中自然生长

## 适用场景

- 个人 AI 的持久化记忆系统
- 需要「时间线感」的知识管理
- 不需要图数据库复杂度的轻量方案
- 有长期对话历史积累的个人知识仓库

## 局限

- 单用户场景设计，无多租户支持
- 语义检索依赖外部 LLM / Embedding 模型
- 字段未形式化定义（工程语言而非学术语言）
